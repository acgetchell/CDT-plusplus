19 [Causal Dynamical Triangulations][CDT] in [C++] uses the
20 [Computational Geometry Algorithms Library][CGAL], [Boost], [TBB], and [Eigen].
21 Arbitrary-precision numbers and functions via [MPFR] and [GMP].
22 Uses [Docopt] to provide a beautiful command-line interface.
23 Uses [Melissa E. O'Neill's Permuted Congruential Generators][PCG] library for high-quality RNGs that pass L'Ecuyer's
24 [TestU01] statistical tests.
25 Uses [Catch] for [BDD]/[TDD].
26 Uses [vcpkg] for library management and building.
27 Uses [Doxygen] for automated document generation.
28 Uses [{fmt}] as a safe and fast alternative to `iostream`.
29

66 ## Getting Started
67 
68 If you just want to run a stable version of the code, grab a versioned [release].
69 
70 Otherwise, you can clone the repo as follows:
71 
72 ~~~
73 git clone https://github.com/acgetchell/CDT-plusplus.git
74 ~~~
75 
76 This will put you on the [development] branch. The project is organized similar to the [PitchFork Layout], as follows:
77 
78 - .github - GitHub specific settings
79 - build - Ephemeral out-of-source build directory
80 - cmake - Cmake configurations
81 - docs - Documentation
82 - external - Includes submodules of external projects (none so far, all using [vcpkg])
83 - include - Header files
84 - scripts - Build, test, and run scripts
85 - src - Source files
86 - tests - Unit tests

88 ### Setup
89 
90 Install [vcpkg]:
91 
92 ```bash
93 git clone https://github.com/Microsoft/vcpkg.git
94 cd vcpkg
95 ./bootstrap-vcpkg.sh
96 ./vcpkg integrate install
97 ```
98 
99 Next, you will need to install up to date versions of [CMake] and [Ninja]. On non-Windows platforms, you will also need
100 `yasm` so that `vcpkg` can install [mpir], which is required for [CGAL].
101 
102 MacOS using [homebrew]:
103 
104 ```bash
105 brew install cmake
106 brew install ninja
107 brew install yasm
108 ```
109 Linux using [apt] (you may also need to install [m4]):
110 ```bash
111 sudo apt-get install cmake
112 sudo apt-get install ninja-build
113 sudo apt-get install yasm
114 sudo apt-get install m4
115 ```
116 
117 At minimum, you need to install prerequisites [Catch], [docopt], [date], [{fmt}], [ms-gsl], [Eigen], [PCG], [tbb], and [CGAL]
118 (which installs [boost], [mpir] and [mpfr]):
119 
120 ```bash
121 vcpkg install catch2
122 vcpkg install docopt
123 vcpkg install date
124 vcpkg install fmt
125 vcpkg install ms-gsl
126 vcpkg install eigen3
127 vcpkg install pcg
128 vcpkg install tbb
129 vcpkg install cgal
130 ```
131 
132 This builds from source, so it will take awhile. To use these successfully, you'll need to
133 set the `CMAKE_TOOLCHAIN_FILE` option in your IDE or whatever invokes [CMake] to wherever
134 you've installed [vcpkg], (e.g. your home directory):
135 
136 ```bash
137 -DCMAKE_TOOLCHAIN_FILE=$HOME/vcpkg/scripts/buildsystems/vcpkg.cmake
138 ```
139 ([Visual Studio 2019] sets this for you by default.)
140 
141 This project uses [C++]17 features, and successfully builds with AppleClang, [gcc-9], [clang-10], and [Visual Studio 2019].
142 On Ubuntu, you may need updated versions of [Clang] or [gcc], and [CMake], which is scripted in [.travis.yml].

144 ### Building
145 
146 If you want to get started right away, in the `scripts` directory run `fast-build.sh` or `fast-build.bat`,
147 depending on your operating system. This will compile the appropriate executables in `RELEASE` mode with no tests.
148 
149 
150 
151 This should result in the main program executable, `cdt` in `build/bin` or `build\Debug`, along with several others.
152 
153 - `cdt-gv` converts output files to [Geomview] format for [visualization](#visualization)
154 - `cdt-opt` is a simplified version with hard-coded inputs, mainly useful for debugging and scripting
155 - `initialize` is used by [CometML] to run [parameter optimization](#parameter-optimization)

157 ## Usage
158 
159 CDT-plusplus uses [Docopt] to parse options from the help message, and so
160 understands long or short argument formats, provided the short argument given
161 is an unambiguous match to a longer one. The help message should be instructive:
162 
163 ~~~text
164 ./build/cdt --help
165 Causal Dynamical Triangulations in C++ using CGAL.
166 
167 Copyright (c) 2014-2019 Adam Getchell
168 
169 A program that generates d-dimensional triangulated spacetimes
170 with a defined causal structure and evolves them according
171 to the Metropolis algorithm. Specify the number of passes to control
172 how much evolution is desired. Each pass attempts a number of ergodic
173 moves equal to the number of simplices in the simulation.
174 
175 Usage:./cdt (--spherical | --toroidal) -n SIMPLICES -t TIMESLICES [-d DIM] -k K --alpha ALPHA --lambda LAMBDA [-p PASSES] [-c CHECKPOINT]
176 
177 Examples:
178 ./cdt --spherical -n 32000 -t 11 --alpha 0.6 -k 1.1 --lambda 0.1 --passes 1000
179 ./cdt --s -n32000 -t11 -a.6 -k1.1 -l.1 -p1000
180 
181 Options:
182  -h --help Show this message
183  --version Show program version
184  -n SIMPLICES Approximate number of simplices
185  -t TIMESLICES Number of timeslices
186  -d DIM Dimensionality [default: 3]
187  -a --alpha ALPHA Negative squared geodesic length of 1-d
188  timelike edges
189  -k K K = 1/(8*pi*G_newton)
190  -l --lambda LAMBDA K * Cosmological constant
191  -p --passes PASSES Number of passes [default: 100]
192  -c --checkpoint CHECKPOINT Checkpoint every n passes [default: 10]
193 ~~~
194 
195 The dimensionality of the spacetime is such that each slice of spacetime is
196 `d-1`-dimensional, so setting `d=3` generates 2 spacelike dimensions and one
197 timelike dimension, with a defined global time foliation. Thus a
198 `d`-dimensional simplex will have some `d-1` sub-simplices that are purely
199 spacelike (all on the same timeslice) as well as some that are timelike
200 (span two timeslices). In [CDT] we actually care more about the timelike
201 links (in 2+1 spacetime) and the timelike faces (in 3+1 spacetime).

203 ## Documentation
204 
205 Online documentation may be found at <https://adamgetchell.org/CDT-plusplus/> automatically generated by [Travis-CI].
206 If you have [Doxygen] installed you can generate the same information
207 locally using the configuration file in `docs\Doxyfile` by simply typing at the top
208 level directory ([Doxygen] will recursively search):
209 
210 ~~~bash
211 doxygen ./docs/Doxyfile
212 ~~~
213 
214 This will generate a `docs/html/` directory containing
215 documentation generated from CDT++ source files. `USE_MATHJAX` has been enabled
216 in [Doxyfile] so that the LaTeX formulae can be rendered in the html
217 documentation using [MathJax]. `HAVE_DOT` is set to **YES** which allows
218 various graphs to be autogenerated by [Doxygen] using [GraphViz].
219 If you do not have GraphViz installed, set this option to **NO**
220 (along with `UML_LOOK`).

222 ## Testing
223 
224 In the `scripts` directory, run `build.sh` or `build.bat` depending on your operating system.
225 
226 Unit tests are run (in `build/tests` or `build\tests\Debug`) via `CDT_test`, the [Catch] executable:
227 
228 ~~~bash
229 ./CDT_test
230 ~~~
231 
232 or (Windows):
233 
234 ~~~cmd
235 CDT_test.exe
236 ~~~
237 
238 You can also run both [CTest] integration and [Catch] unit tests in the `build` directory with:
239 
240 ~~~bash
241 cmake --build . --target test
242 ~~~
243 
244 or (Windows):
245 
246 ~~~
247 ctest -C Debug
248 ~~~
249 
250 In addition to the command line output, you can see detailed results in the
251 `build/Testing` directory which is generated thereby.
252 
253 Unit tests are turned off with `-D ENABLE_TESTING:BOOL=FALSE`, e.g. `scripts/fast-build.sh`.

289 ## Parameter Optimization
290 
291 [CometML] is used to record [Experiments] which conduct [Model Optimization]. The script to do
292 this is `optimize-initialize.py`. In order for this to work, you must install the following
293 into your Python [virtual environment].
294 
295 ~~~bash
296 pip install tensorflow
297 pip install comet-ml
298 ~~~
299 
300 You can then run experiments and look at results on https://www.comet.ml!

302 ## Visualization
303 
304 [Geomview] is used to generate pictures of triangulations using the `cdt-gv` binary. In order for this to
305 work, you must have [Geomview] installed (which doesn't work on Windows). On MacOS:

307 ~~~bash
308 brew cask install xquartz
309 brew install geomview
310 ~~~
