of testing for the CDT de Sitter C-phase.

Work incrementally. Preserve all existing 2+1D functionality and tests.

1. Fix the existing Metropolis implementation

Before implementing 4D evolution:

- Fix the 3D SIX_TWO action update. It must subtract four from N3_31_13, not
  from N3_22.
- Remove CalculateA1 based on historical move frequencies.
- Implement the actual Metropolis-Hastings proposal ratio:

  acceptance = min(1, exp(-delta_action) *
                       reverse_proposal_probability /
                       forward_proposal_probability)

- Propose and execute one move at a time.
- Do not queue moves that were accepted against an old geometry.
- Construct a candidate state, validate it, calculate its complete action, and
  only then accept or reject it.
- A rejected or invalid move must leave the original state unchanged.
- Use one persistent seeded RNG per simulation.
- Add --seed to command-line programs.
- Add tests showing that the same seed produces identical results.
- Add a quadratic volume-fixing term:

  delta_S_volume = epsilon * (N - target_N)^2

- Verify incremental action differences against complete action recomputation.
- Add move-plus-inverse tests and numerical detailed-balance tests on small
  triangulations.
